# PyQt5 OrinNano Manual Style Guide

## Language and Audience

- Target language: `en-001` international English.
- Audience: technical learners, instructors, and users of the OrinNano AI experiment platform.
- Tone: clear, formal, instructional, and suitable for a technical lab manual.

## Translation Rules

- Preserve facts, safety requirements, operation order, numeric values, units, version numbers, command syntax, file paths, variables, API names, and code identifiers.
- Use concise imperative wording for procedures, for example "Click", "Open", "Run", "Select", and "Configure".
- Use "must" for mandatory safety and operating requirements.
- Use "may" only for optional actions or possible outcomes.
- Keep product and technical names unchanged unless translating surrounding explanatory text.

## Fixed Terms

- `OrinNano` remains `OrinNano`.
- `PyQt5` remains `PyQt5`.
- `QtDesigner` remains `QtDesigner`.
- `Anaconda3` remains `Anaconda3`.
- `TensorFlow 2` remains `TensorFlow 2`.
- `Jetson` remains `Jetson`.
- `实验手册` translates as `Lab Manual`.
- `安全提示` translates as `Safety Instructions`.

## Formatting and Localisation

- Keep document structure, heading hierarchy, tables, captions, numbering, images, drawings, footer text, and field structure as close to the source as possible.
- Do not localise or redraw Chinese text inside images and screenshots. Record this as a QA risk.
- Avoid region-specific spelling where possible. Use neutral technical English.
- Do not leave `TODO(...)`, translator notes, placeholders, or machine-translation notices in final deliverables.
